## 2. Page Template Structure
To avoid thin/duplicate content penalties from Google, each `/tailor/[job-title]` page will dynamically load unique copy regarding specific ATS triggers for that role.

**URL Route:** `ats-hacker-swart.vercel.app/tailor/software-engineer`

**Template Components:**
1. **Dynamic H1:** "Pass the ATS for [Job Title] Roles"
2. **Dynamic Subtitle:** "Don't let a robot reject your [Job Title] resume. We rewrite your resume to semantically match exactly what Applicant Tracking Systems look for in [Job Title] candidates."
3. **The 'Why It Matters' Block:** A dynamic text block (generated by OpenAI beforehand) explaining *why* ATS systems filter out [Job Title]s (e.g., "For Software Engineers, Workday and Greenhouse aggressively scan for specific framework versions like React 18 or Node.js. If your formatting breaks the parser, you are auto-rejected.")
4. **Top 5 Must-Have Keywords:** A dynamically injected list of 5 standard skills expected for that role, proving we understand their niche.
5. **The Interactive Demo/Upload Component:** Our standard `$5.00` checkout and PDF upload block.
6. **FAQ Section:** Role-specific FAQs (e.g., "How does ATS Hacker help Project Managers specifically?").

### Proposed Code Structure (`src/app/tailor/[slug]/page.tsx`)
```tsx

import { UploadCloud, CheckCircle } from 'lucide-react';
import { getJobData } from '@/lib/seo-data'; // A local JSON file containing the 50 roles & copy

export async function generateMetadata({ params }: { params: { slug: string } }) {
  const job = getJobData(params.slug);
  return {
    title: `Optimize your ${job.title} Resume for ATS | ATS Hacker`,
    description: `Stop getting auto-rejected. We rewrite your ${job.title} resume to perfectly match job descriptions and beat Applicant Tracking Systems.`,
  };
}

export default function TailoredLandingPage({ params }: { params: { slug: string } }) {
```
